# 🚀 Neural Engine: Knowledge Graph Visualizer

## 🧠 Thinking Behind the Project

As a Frontend Developer comfortable with React, jumping into this project was both exciting and challenging. This was my **first time using Next.js and React Flow**, and it pushed me beyond my comfort zone.

### ⚡ The Struggle

Moving from standard React components to a **node-based graph system** felt like learning from scratch.  

I spent hours understanding:
- How **Handles** work in React Flow  
- Why edges weren’t connecting  
- Debugging issues like **z-index conflicts** and incorrect handle types  

After breaking things multiple times and fixing them again, I finally built something that feels like a **sci-fi inspired interface**.

---

## 🛠️ Tech Stack

- **Framework:** Next.js 
- **Graph Engine:** React Flow (`@xyflow/react`)
- **State Management:** Zustand
- **Styling:** Tailwind CSS
- **Icons:** Lucide React

---

## ✨ Features

- 🌌 **Futuristic UI**  
  Neural network-inspired design with glowing nodes and radar effects  

- 🧩 **CRUD Operations**  
  - Create nodes  
  - Update node content via sidebar  
  - Delete nodes  

- 🗺️ **Google Maps Style Navigation**  
  - MiniMap for large graphs  
  - Smooth pan & zoom controls  

- 📱 **Responsive Design**  
  Works across desktop and mobile devices  

- 📊 **Live Metrics**  
  Real-time calculation of active nodes and system telemetry  

---

## 🧠 What I Learned

### 🚀 Next.js is Powerful
File-based routing and performance optimizations are game-changers compared to plain React.

### 🔄 State Management is Hard
Handling graph data (nodes + edges) is much more complex than simple apps. Zustand made it manageable.

### 🎨 UI/UX Matters
The radar and glow effects weren’t just design choices—they were about creating an **immersive experience**.
